title: beaker.autobase
description: Multi-writer Collaborative Drives backed by Autobase.
---

The Autobase API provides multi-writer Collaborative Drives. Multiple Writers can append to a Collaborative Drive; reads are linearised across all writers. The API shape mirrors `beaker.hyperdrive` so template authors work with a familiar interface.

See the [Collaborative Drives guide](/docs/api/advanced/collaborative-drives/) for a full walkthrough.

## API

### beaker.autobase.collaborativeDrive(url)

Create a scoped Collaborative Drive instance. Methods on the instance accept paths rather than full URLs.

```javascript
var drive = beaker.autobase.collaborativeDrive('hyper://abc123../')
await drive.list('/')
```

Returns a **CollaborativeDrive** instance with all the read/write methods below, plus writer management methods scoped to this drive.

---

### beaker.autobase.createCollaborativeDrive(\[opts\])

Create a new Collaborative Drive. By default this opens the create-drive modal so the user can review or edit the title and description. Pass `prompt: false` to skip the modal and create immediately (useful when building your own UI, e.g. a template page).

* **opts** Object.
  * **title** String.
  * **description** String.
  * **type** String. e.g. `"walled.garden/forum"`
  * **prompt** `false`. Set to `false` to bypass the modal and create the drive immediately.
* Returns **Promise&lt;CollaborativeDrive&gt;**.

```javascript
// Opens the create-drive modal (default)
var drive = await beaker.autobase.createCollaborativeDrive({ title: 'My Forum' })
console.log(drive.url) // hyper://...

// Skip the modal — create directly
var drive = await beaker.autobase.createCollaborativeDrive({ title: 'My Forum', prompt: false })
```

## Write Methods

Write methods append an operation to the Autobase log. The operation is linearised and applied to all peers.

Only Writers can call write methods. Non-writers receive an `ArchiveNotWritableError`.

## Writer Management

### drive.createInvite(\[opts\])

Generate an invite URL that others can use to request write access.

* **opts.multiUse** Boolean. If true, the invite can be claimed by multiple people (default: false).
* Returns **Promise&lt;String&gt;** — the invite URL.

```javascript
var inviteUrl = await drive.createInvite({ multiUse: true })
// Share this URL with the person you want to invite
```

### beaker.autobase.claimInvite(inviteUrl\[, opts\])

Claim an invite URL to request write access. Called by the recipient.

* **inviteUrl** String. The invite URL generated by `createInvite`.
* **opts.profileUrl** String. Your Profile Drive URL, so the owner can see who you are.
* Returns **Promise&lt;{ writerKey }&gt;**.

```javascript
var { writerKey } = await beaker.autobase.claimInvite(inviteUrl, {
  profileUrl: 'hyper://my-profile-key/'
})
```

### beaker.autobase.requestAccess(url\[, opts\])

Request write access to a Collaborative Drive without an invite (stranger-initiated).

* **url** String. The Collaborative Drive URL.
* **opts.profileUrl** String. Your Profile Drive URL.
* Returns **Promise&lt;{ writerKey }&gt;**.

### drive.listRequests()

List pending writer requests.

* Returns **Promise&lt;Array&lt;{ writerKey, profileUrl, requestedAt }&gt;&gt;**.

```javascript
var requests = await drive.listRequests()
for (const req of requests) {
  console.log(req.writerKey, req.profileUrl)
}
```

### drive.watchRequests(\[onRequest\])

Watch for writer-access requests arriving over the network. Emits a `changed` event each
time a new request is received, so an owner's UI can refresh `listRequests()` live instead
of polling.

* **onRequest** Function (optional). Called on each `changed` event.
* Returns an **EventTarget** that emits `changed`.

```javascript
drive.watchRequests(async () => {
  var requests = await drive.listRequests()
  // re-render your approve/deny UI
})
```

### drive.approveRequest(writerKey\[, opts\])

Approve a pending writer request. Only the drive owner can call this.

* **writerKey** String.
* **opts.profileUrl** String. Optional override for the profile URL to store.

### drive.denyRequest(writerKey)

Deny and remove a pending writer request.

### drive.removeWriter(writerKey)

Remove an approved writer from the drive. Only the drive owner can call this.

### drive.listWriters()

List all approved writers and their profile URLs.

* Returns **Promise&lt;Array&lt;{ writerKey, profileUrl }&gt;&gt;**.

```javascript
var writers = await drive.listWriters()
for (const w of writers) {
  var profile = await beaker.hyperdrive.drive(w.profileUrl).getInfo()
  console.log(profile.title, w.writerKey)
}
```
